## Связный список: линейная структура из узлов

### Описание

Связный список — это линейная структура данных, состоящая из узлов (nodes), cоединённых ссылками. Каждый узел обычно хранит два поля:
- `data` содержит значение узла;
- `next` содержит ссылку в виде указателя на следующий узел.

Первый узел списка называют головой (`head`). А последний узел, хвост (`tail`) указывает на специальное значение (например `null`), обозначая конец списка. Можно представить связный список как цепочку:
```text
[data | next] <- head
		↓
[data | next]
		↓
[data | next] <- tail
		↓
		null
```

Если в массиве элементы лежат **подряд** в памяти, и мы можем обратиться к элементу по индексу за постоянное время, то в связном списке элементы могут лежать **в разных местах памяти**, а порядок задаётся через `next`. Отсюда следуют ключевые свойства:
- Доступ по индексу медленный. Чтобы попасть в узел с индексом `i`, нужно пройти все узлы от головы до этого места. Это даёт линейное время `O(n)` в операциях доступа к произвольному элементу.
- Но вставка и удаление по ссылке — "дешёвые". Если у нас уже есть ссылка на узел, мы можем “вклинить” новый элемент или убрать существующий, просто поменяв пару ссылок, без сдвига всех остальных элементов.

Чаще всего определяют два вида реализации: одно и двух связный список. В односвязном списке каждый узел хранит указатель на следующий узел. В двусвязном списке каждый узел хранит указатели на предыдущий и на следующий узел.

## Связный список против массива

Кратко сравним массивы и связные списки на уровне абстракции. Связный список часто приводят как пример структуры, которая позволяет выполнять операции вставки и удаления в произвольное место за постоянное время, гарантируя сложность `O(1)`. Для этого необходимо иметь ссылку на нужное место. Однако, доступ к элементу на практике требует пройти список от начала давая сложность `O(n)`. Для массива наоборот, вставка и удаление в середине требуют перезаписывания всего массива, но доступ по индексу позволяет обращаться к элементам за `O(1)`. 

В таком сравнении можно утверждать, что массив выбирают, когда важно быстро обращаться к элементам по позиции. А связный список когда важнее часто вставлять и удалять элементы, а линейный проход по структуре приемлем.

В реальной практике большинство языков и библиотек оптимизируют массивы настолько хорошо, что связные списки проигрывают им даже там, где теоретически должны выигрывать. Поэтому связный список стоит воспринимать как учебную модель, которая помогает:
- понять работу указателей и ссылок;
- отработать паттерны обхода и модификации списков;
- подготовиться к задачам, которые встречаются на собеседованиях.

## Реализации и модели в популярных языках

Перейдём к реализациям этой структуры данных в популярных языках программирования.
### Python

В Python нет встроенного связного списка, поэтому напишем свой класс `Node` и `LinkedList`.
```python
class Node:
    def __init__(self, value):
        self.value = value
        self.next = None


class LinkedList:
    def __init__(self):
        self.head = None

    # Добавление в начало, в конец, поиск, удаление и т.д.
```
### Go

В Go есть пакет `container/list`, который реализует двусвязный список. Тип `list.List` хранит узлы и каждый элемент — это `*list.Element`, содержащий `Value` и ссылки на соседей.

### C++

В C++ стандартная библиотека предлагает односвязный список `std::forward_list` и  двусвязный список `std::list`. В них каждый элемент представляет собой узел с данными и ссылками. Итераторы позволяют проходить список вперёд (и назад для `std::list`). Доступ по индексу линейный, зато вставка/удаление имеют постоянную сложность.
### Rust

В Rust стандартная библиотека содержит двусвязный список `std::collections::LinkedList<T>`. Однако документация и сообщество Rust подчёркивают, что `Vec` и `VecDeque` почти всегда предпочтительнее по производительности и простоте. Тип `LinkedList` является скорее учебным инструментом и редко используется на практике.

## Основные операции в связном списке

Рассмотрим на абстрактном уровне типичные операции, которые будут встречаться в задачах:
1. Обход (traversal) списка:
   - начинаем с `head`;
   - последовательно переходим по `next`, пока не дойдём до конца, обычно это `null`;
   - во время обхода можем печатать значения, считать длину, искать элемент и т.д.
2. Вставка в начало:
   - создаём новый узел;
   - добавляем в него указатель на текущее начало списка `new.next = head`;
   - обновляем текущее начало списка `head = new`.
3. Вставка в середину (после заданного узла):
   - находим узел `prev`, после которого хотим вставить;
   - сохраняем указатель на следующий узел `new.next = prev.next`;
   - обновляем указатель на следующий узел `prev.next = new`.
4. Удаление узла:
   - находим узел `prev` перед удаляемым;
   - обновляем указатель на следующий узел `prev.next = prev.next.next`.

## Где используются связные списки

С практической стороны связные списки используются не так часто, как массивы или динамические массивы. Многие их теоретические преимущества нивелируются особенностями современных процессоров, кешей и реализаций стандартных контейнеров. Но есть решения в реальных системах где связные списки являются подходящим решением:
- undo/redo или навигация, например в браузере(кнопка "назад" и "вперёд"). В таком сценарии доступ к конкретному индексу нужен редко. История изменений хранится как список состояний и последовательный проход по истории является частой задачей;
- внутренние структуры в ОС, БД, сетевых буферах иногда реализуются на связных списках для решения специфичных задач;
- и наконец, реализация списков соседей в графах или цепочек в хэш-таблицах.

С учебной стороны и собеседований связный список может быть подходящей структурой для задач на указатели. Например:
- разворот списка;
- удаление из середины;
- алгоритм быстрого и медленного указателя с конкретной логикой, например поиск цикла;

## Задача: поиск цикла в связном списке

Рассмотрим классическую задачу для связного списка — поиск цикла. Это один из самых популярных примеров использования медленного и быстрого указателя.
### Условие задачи

Условие задачи звучит так:

> Дан односвязный список. Нужно определить, содержит ли он цикл. То есть существует ли такой узел, в который можно попасть, двигаясь по `next`, из разных узлов.

Обычный список заканчивается `null` проход по `next` рано или поздно упирается в конец. В списке с циклом, двигаясь по `next`, мы будем ходить по кругу бесконечно. Можно было бы хранить множество уже посещённых узлов (например, через хэш-таблицу) и на каждом шаге проверять, не видели ли мы этот узел раньше. Это работает за `O(n)` по времени, но требует дополнительной памяти `O(n)`. Но существует алгоритм Флойда, который использует два указателя и не требует дополнительной памяти.
### Идея алгоритма (медленный и быстрый указатели)

В алгоритме Флойда мы используем два указателя медленный `slow` двигается по одному шагу за раз, то есть на каждом шаге `slow = slow.next`. А быстрый `fast` двигается по два шага за раз, то есть на каждом шаге `fast = fast.next.next`. Возможны два случая:
1. Цикла нет. В какой-то момент `fast` или `fast.next` станет `null`, значит, список закончился, и цикла нет.
2. Цикл есть. В какой то момент быстрый и медленный указатель встретятся на одном и том же узле. Это будет означать, что в списке есть цикл.
### Реализация на Python

Посмотрим на реализацию на Python. Предположим, у нас есть класс узла:

```python
class Node:
    def __init__(self, value):
        self.value = value
        self.next = None
```

Функция проверки цикла будет выглядеть так:
```python
def has_cycle(head: Node) -> bool:
    slow = head
    fast = head

    while fast is not None and fast.next is not None:
        slow = slow.next
        fast = fast.next.next

        if slow is fast:
            return True

    return False
```

Алгоритм работает за линейное время `O(n)` — каждый указатель проходит по списку ограниченное число раз. Дополнительная память `O(1)` — мы используем только несколько ссылок, не храним множество посещённых узлов.

Зафиксируем определение:

> Связный список — это линейная структура данных, состоящая из узлов, каждый из которых хранит значение и ссылку на следующий узел (а в двусвязном варианте — ещё и на предыдущий).

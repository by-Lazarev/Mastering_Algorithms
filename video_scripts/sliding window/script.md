# Скользящее окно

## Введение: зачем нужен алгоритм "скользящее окно"

В этом видео мы разберём один из алгоритмический паттерн “скользящее окно”. Его можно назвать подвидом для алгоритма двух указателей. Он позволяет заменить вложенные циклы одним проходом по данным и перейти от квадратичной сложности `O(n^2)` к линейной `O(n)`.

В этом видео мы:
- разберём, что такое “скользящее окно”;
- рассмотрим два основных варианта: окно фиксированного размера и окно переменного размера;
- разберём задачу на сумму подмассива с использованием скользящего окна;
- и научимся узнавать задачи, для которых эффективное решение основано на этом алгоритмическом паттерне.

---
## Введение

Начнём с примера: представим, что у нас есть массив чисел и нужно найти **максимальную сумму подмассива длины `k`**. В наивное решение мы перебираем все отрезки длины `k` и для каждого отрезка пересчитываем сумму с нуля, затем сравниваем и запоминаем максимум. Реализация потребует два вложенных цикла, внешний по старту подмассива и внутренний по элементам внутри этого подмассива. Сложность такого решения — `O(n * k)` и в худшем случае это почти `O(n^2)`.

При использовании алгоритма скользящего окна мы считаем сумму первого окна. Дальше, двигая окно на один элемент, переиспользуем уже посчитанную сумму  вычитая элемент, который вышел из окна, и добавляя новый. Вместо пересчёта каждого окна с нуля, мы делаем по одной операции обновления на шаг. Это даёт нам линейную сложность `O(n)`.

---
## Вариант 1: скользящее окно фиксированного размера

В первом варианте реализации, который мы разберём скользящее окно имеет фиксированный размер `k`. Окно представляет собой текущий подмассив `nums[left:right]`, внутри которого мы поддерживаем какую-то агрегированную величину: сумму, максимум, количество элементов и т.д.
### Абстрактная схема
На уровне абстракции окно имеет фиксированную длину `k`. Вначале мы считаем сумму первых `k` элементов — это стартовое окно. Затем двигаем окно на один шаг вправо:
- вычитаем `nums[left]` — элемент, который вышел из окна;
- добавляем `nums[right]` — новый элемент, который вошёл в окно;
- сдвигаем `left` и `right` на один;
- на каждом таком шаге обновляем ответ (например, максимальную сумму).

---
## Вариант 2: скользящее окно переменного размера

Во втором варианте реализации алгоритма размер окна не фиксирован и зависит от условий задачи. Мы так же работаем с двумя указателями, правый расширяет окно вправо, добавляя новые элементы, а левый сжимает окно слева, когда оно перестаёт удовлетворять условию.

Этот вариант реализации подходит для задач, в нужно найти подмассив соответсвующий условию. Это может быть как минимальный, так и максимальный по длине подмассив или подстрока. А условием может быть ограничение, например: сумма элементов, целевое значение или список символов и т.д. Схема решения обычно выглядит так:
1. Изначально `left = 0`, `right = 0` и окно соответственно пустое.
2. Двигаем `right` вправо, добавляя элементы и обновляя состояние (сумму, частоты символов).
3. Как только условие перестаёт выполняться или становится двигаем `left` вправо, убирая элементы из окна.
4. На каждом шаге, когда окно валидно, обновляем ответ: длину, сумму, позицию.

---
## Дополнение

Мы определяем скользящее окно как отдельный алгоритм от алгоритма с двумя указателями по нескольким причинам:
1. В скользящем окне нас интересует вся подстрока между указателями, `left` и `right` вместе описывают текущий отрезок данных, а не просто отдельные точки;
2. Во время выполнения алгоритма мы поддерживаем агрегат по всему окну сохраняя сумму элементов, их количество, или множество уникальных значений.

---
## Где используется скользящее окно

Скользящее окно — это алгоритм для задач, где явно фигурируют **подмассивы или подстроки** и где важно эффективно пересчитывать агрегат на соседних окнах. В таких задачах мы получаем линейная структуру, например массив или строку. В условии обычно говорится о “окне”, “отрезке”, “подстроке”, “подмассиве”. В этой подструктуре нужно найти длину наибольшего или наименьшего подмассива с определённым свойством. Это может быть максимум или минимум суммы на отрезке или количество подмассивов, удовлетворяющих условию.

## Задача: максимальная сумма подмассива фиксированной длины (Python)

Разберём пример задачи с решением основаным на окне фиксированного размера.
### Условие задачи
По условию задачи у нас массив целых чисел `nums` и число `k`. Нужно найти максимальную сумму подмассива длины `k`.

В наивной реализации для каждого возможного старта от `i = 0` до `i = n - k` мы бы пересчитывали сумму `k` элементов `nums[i : i + k]` и обновлять максимум. Сложность такого решения будет `O(n * k)` или близкой к `O(n ^ 2)`.

---
### Решение со скользящим окном (Python)

В реализации со скользящим окном считаем сумму первых `k` элементов — это наше стартовое окно. Затем двигаем окно по одному элементу вправо:
- вычитаем элемент, который выходит слева;
- добавляем элемент, который входит справа;
- обновляем максимум.

```python
def max_subarray_sum_k(nums, k):
    n = len(nums)
    window_sum = sum(nums[0:k])
    max_sum = window_sum

    for right in range(k, n):
        left = right - k
        window_sum = window_sum - nums[left] + nums[right]
		max_size = max(max_sum, window_sum)

    return max_sum


nums =[2, 1, 5, 1, 3, 2]
k = 3

print(max_subarray_sum_k(nums, k))  # 9
```

---
### Разбор по шагам

Пусть `nums = [2, 1, 5, 1, 3, 2]` и `k = 3`.
1. Стартовое окно `[2, 1, 5]`:
   - `window_sum = 2 + 1 + 5 = 8`;
   - `max_sum = 8`.
2. Сдвигаем окно на один вправо:
   - новое окно `[1, 5, 1]`;
   - вычитаем `2`, добавляем `1`: `window_sum = 8 - 2 + 1 = 7`;
   - `max_sum` остаётся `8`.
3. Следующий сдвиг:
   - окно `[5, 1, 3]`;
   - вычитаем `1`, добавляем `3`: `window_sum = 7 - 1 + 3 = 9`;
   - `max_sum = 9`.
4. Ещё один сдвиг:
   - окно `[1, 3, 2]`;
   - вычитаем `5`, добавляем `2`: `window_sum = 9 - 5 + 2 = 6`;
   - `max_sum` остаётся `9`.

Мы прошли массив один раз, каждый раз обновляя сумму за `O(1)` операций, и получили итоговую сложность `O(n)`.

## Заключение урока

Давайте подведём итог. В этом видео мы разобрали алгоритм “скользящее окно” — паттерн, который помогает работать с подмассивами и подстроками без вложенных циклов.

Мы рассмотрели два основных варианта реализации этого паттерна:
- **Фиксированное окно**, в котором размер окна заранее известен, мы двигаем его по массиву и поддерживаем агрегат (сумму, максимум и т.д.), обновляя результат за `O(1)`.
- **Переменное окно**, в котором границы окна определяются условиями задачи, мы расширяем окно вправо и сжимаем слева, поддерживая внутри состояние (сумма, частоты символов, количество уникальных элементов и т.п.).

В итоге, скользящее окно — это частный случай подхода с двумя указателями, но с акцентом на **все элементы внутри окна**, а не только на значения по указателям. Если ваше решение решение заключается в пересчёте каждого подмассива или подстроки с нуля, то вероятно можно описать текущее окно двумя указателями и обновлять результат инкрементально, передвигая окно по данным.

## Стек: структура с доступом LIFO
### Описание
Начнём со стека. Стек — это структура данных, в которой элементы добавляются и удаляются только с одного конца. Этот конец обычно называют вершиной стека.

Главный принцип стека — **LIFO**, то есть *last in, first out*: последним пришёл, первым вышел. Если мы положили в стек элементы `10`, `20`, `30`, то первым обратно получим `30`, потом `20` и потом `10`. Интуитивно стек можно представить как стопку тарелок. Новые тарелки кладутся сверху, и снимать мы тоже можем только верхнюю. Нельзя достать тарелку из середины, не сняв сначала те, что лежат сверху.

У стека есть несколько базовых операций:
- `push` — положить элемент на вершину;
- `pop` — снять верхний элемент;
- `top` или `peek` — посмотреть верхний элемент, не удаляя его;
- `is_empty` — проверить, пуст ли стек.

На уровне абстракции стек определяет строгий способ доступа к значениям. Мы не выбираем индекс, как в массиве, и не можем получать значения по ключу, как в хэш-таблице. Мы всегда работаем только с последним добавленным элементом. Поэтому стек подходит для задач, где есть:
- вложенность;
- обратный порядок обработки;
- необходимость “вернуться назад” по уже выполненным шагам.
### Реализации
Посмотрим, как стек реализован в популярных языках. В любом языке можно использовать обычный массив, добавлять и удалять элементы с конца массива позволяет любая реализация.
#### Python
В Python можно использовать готовую реализацию стека из стандартной библиотеки — `queue.LifoQueue`. `LifoQueue` работает по принципу *last in*, *first out*, то есть реализует именно стек.
```python
from queue import LifoQueue

stack = LifoQueue()

stack.put(10)   # push
stack.put(20)
stack.put(30)

top = stack.get()   # pop -> 30
print(top)          # 30

stack.put(40)

next_top = stack.get()  # pop -> 40
print(next_top)         # 40
```

Стандартный интерфейс `LifoQueue` не поддерживает просмотр верхнего элемента без удаления, но есть множество других реализаций, которые позволяют посмотреть последний элемент и предоставляют другие возможности.
#### Go
В Go нет встроенного типа `stack`, но идиоматичная реализация — структура поверх слайса.
```go
// Go — stack

type Stack struct {
    items []int
}

func (s *Stack) Push(value int) {
    s.items = append(s.items, value)
}

func (s *Stack) Pop() (int, bool) {
    if len(s.items) == 0 {
        return 0, false
    }
    n := len(s.items) - 1
    value := s.items[n]
    s.items = s.items[:n]
    return value, true
}

func (s *Stack) Peek() (int, bool) {
    if len(s.items) == 0 {
        return 0, false
    }
    return s.items[len(s.items)-1], true
}

func (s *Stack) IsEmpty() bool {
    return len(s.items) == 0
}

func (s *Stack) String() string {
    return fmt.Sprint(s.items)
}

func exampleStack() {
    var stack Stack

    stack.Push(10)
    stack.Push(20)
    stack.Push(30)

    print(stack) // [10 20 30]

    top, _ := stack.Peek()
    print(top)   // 30

    value, _ := stack.Pop()
    print(value) // 30
    print(stack) // [10 20]
}
```
#### C++
В C++ есть `std::stack`, который прямо реализует LIFO-структуру.

```cpp
#include <stack>

void example_stack() {
    std::stack<int> stack;

    stack.push(10);
    stack.push(20);
    stack.push(30);

    print(stack);        //  — абстрактный вывод[3][1][2]

    int top = stack.top();
    print(top);          // 30

    stack.pop();

    print(stack);        //[1][2]
}
```

Здесь `std::stack` по умолчанию использует `std::deque` как внутренний контейнер,  
но на уровне абстракции нас интересуют только операции `push`, `pop`, `top` и `empty`.
#### Rust
В Rust нет отдельного типа `Stack`, но стандартный `Vec<T>` подходит для реализации стека: `push` добавляет элемент в конец, а `pop` снимает последний элемент.
```rust
struct Stack<T> {
    items: Vec<T>,
}

impl<T> Stack<T> {
    fn new() -> Self {
        Stack { items: Vec::new() }
    }

    fn push(&mut self, value: T) {
        self.items.push(value);
    }

    fn pop(&mut self) -> Option<T> {
        self.items.pop()
    }

    fn peek(&self) -> Option<&T> {
        self.items.last()
    }

    fn is_empty(&self) -> bool {
        self.items.is_empty()
    }
}

fn example_stack() {
    let mut stack = Stack { items: vec![] };

    stack.push(10);
    stack.push(20);
    stack.push(30);

    print(&stack.items);          //[2][3][1]

    if let Some(top) = stack.peek() {
        print(top);               // 30
    }

    if let Some(value) = stack.pop() {
        print(value);             // 30
    }

    print(&stack.items);          //[1][2]
}
```

Во всех четырёх языках мы получили одну и ту же абстракцию: тип `Stack` с методами `push`, `pop`, `is_empty`, `peek` (кроме Python). Во всех примерах стек работает по принципу LIFO, независимо от деталей реализации.
### Где используется стек
Разберём задачи которые используют стек:

Во-первых, стек естественно возникает там, где есть **вложенность**.
Например проверка корректности скобок. Когда мы видим открывающую скобку, мы кладём её в стек. А когда видим закрывающую, мы должны снять последнюю добавленую в стек и проверить, совпадает ли тип скобок.

Во-вторых, стек полезен там, где есть **обратный порядок обработки**.
Например, если мы хотим откатить действия назад: последнее действие должно отменяться первым. К этому же типу можно отнести графовые алгоритмы. Например, глубинный обход DFS можно реализовать рекурсией, а можно явно держать стек вершин, которые ещё нужно обработать.

В-третьих, стек используется при вычислении выражений:
- постфиксная и инфиксная запись;
- разбор выражений;
- парсинг.

Напомню, что стек существует не только как структура в алгоритмической теории, но и как часть выполнения программы в *computer science*. Когда одна функция вызывает другую, язык и среда выполнения обычно используют **стек вызовов**. В стеке будут храниться параметры функций, локальные переменные, адрес возврата и временные данные.
### Заключение

Зафиксируем определение:

> Стек — это структура данных, в которой элементы обрабатываются по принципу “последним пришёл — первым вышел”, поэтому она хорошо подходит для вложенности, отката действий и обратного порядка обработки.

## Куча: структура с приоритетом
### Описание
Теперь перейдём к структуре данных "куча". Как и в случае со стеком мы рассмотрим кучу как структуру данных **heap**, которая используется для реализации **priority queue** (очередь с приоритетом). В *computer science* словом **куча** также называют область памяти для динамического выделения объектов. Например, при запуске программы для неё выделяется куча, в которой будут хранится данные программы.

Если стек отвечает на вопрос “какой элемент был добавлен последним”, то куча отвечает на вопрос: **какой элемент сейчас имеет наивысший приоритет**. Таким образом куча — это структура данных, в которой мы можем эффективно получать минимальный или максимальный элемент, в зависимости от того используем мы **min-heap** или **max-heap**.

Основные операции для кучи:
- `insert` — добавить элемент;
- `peek` / `top` — посмотреть лучший элемент;
- `extract_min` или `extract_max` — достать лучший элемент и удалить его.

Верхний элемент в куче всегда “лучший” по приоритету, но остальные элементы могут располагаться в любом порядке. Обычно куча реализуется как бинарное дерево, но хранится не как узлы и указатели, а внутри массива. Для любой вершины её левым потомком будет значение с индексом `2*i + 1`, а правым с индексом `2*i + 2`.

Разберём абстракцию подробнее. Проведём несколько операций над структурой и посмотрим как она будет изменяться. В итоге сравним визуальное отображение дерева и расположение элементов в массиве. 

```
heap = []
heappush(heap, 30)
heap -> [30]
heappush(heap, 20)
# Добавим в конец -> [30, 20]
# Сравним с (i - 1) // 2 -> значением по индексу 0
# Т.к. 20 < 30 меняем их местами
heap -> [20, 30]
heappush(heap, 10)
# Добавим в конец -> [20, 30, 10]
# Сравним с (i - 1) // 2 -> значением по индексу 0
# Т.к. 10 < 20 меняем их местами
heap -> [10, 30, 20]
# В виде дерева:
        10
      /    \
    30      20

# На этом этапе heap мог быть и [10, 20, 30]
# В виде дерева:
        10
      /    \
    20      30
```
### Реализации
Теперь посмотрим, как куча представлена в популярных языках.

В Python используется модуль `heapq`. Он работает поверх обычного списка и реализует **min-heap**, то есть на вершине всегда минимальный элемент.
```python
import heapq

# Python — heap (min-heap)

heap = []

heapq.heappush(heap, 30)
heapq.heappush(heap, 10)
heapq.heappush(heap, 20)

print(heap)          

top = heapq.heappop(heap)
print(top)
print(heap)
```

В Go есть пакет `container/heap`, который предоставляет функции `heap.Init`, `heap.Push`, `heap.Pop` и другие. Чтобы им пользоваться, необходимо описать свой тип кучи — например, `IntHeap` для целых чисел — и сделать его совместимым с `container/heap`.  
Детали того, как именно устроен `IntHeap`, мы здесь опустим, важна сама идея использования
```go
import "container/heap"

// считаем, что IntHeap уже реализован и совместим с container/heap
// под капотом там хранится слайс чисел и поддерживается свойство кучи

func exampleHeap() {
    h := &IntHeap{}   // создаём пустую кучу
    heap.Init(h)      // инициализируем её

    heap.Push(h, 30)
    heap.Push(h, 10)
    heap.Push(h, 20)

    print(*h)         // внутренняя структура кучи, порядок не обязательно отсортированный

    minValue := heap.Pop(h).(int)  // извлекаем минимальный элемент
    print(minValue)   // 10

    print(*h)         // куча после удаления минимума
}
```
>  Функции из `container/heap` гарантируют, что после каждого `Push` и `Pop` внутренняя структура остаётся корректной min-heap.

В C++ стандартная структура для этого — `std::priority_queue`. По умолчанию она работает как **max-heap**, то есть наверху будет максимальный элемент.
```cpp
#include <queue>
#include <vector>
#include <functional>

void example_heap() {
    std::priority_queue<int, std::vector<int>, std::greater<int>> heap;

    heap.push(30);
    heap.push(10);
    heap.push(20);

    print(heap.top()); // 10

    heap.pop();

    print(heap.top()); // 20
}
```

В Rust используется структура `BinaryHeap`. По умолчанию она работает как **max-heap**, то есть наверху будет максимальный элемент.
```rust
use std::cmp::Reverse;
use std::collections::BinaryHeap;

fn example_heap() {
    let mut heap = BinaryHeap::new();

    heap.push(30);
    heap.push(10);
    heap.push(20);

    print(&heap);

    let top = heap.pop().unwrap();
    print(top.0);

    print(&heap);
}
```
### Где используется куча
Разберём задачи которые используют кучу:

Во-первых, задачи на реализацию **priority queue** - приоритетной очереди. Это может быть работа со списоком с событий, таймеров, запросов или процессов. В них нам нужно каждый раз брать следующий “самый важный” элемент. Так же куча удобна при слиянии нескольких отсортированных потоков данных. Например, если у нас есть несколько отсортированных списков, мы можем держать их текущие минимумы в куче и каждый раз доставать наименьший из них.

Во-вторых, куча используется в задачах на поиск `k` минимальных, максимальных или часто встречающихся элементов. Сюда же можно отнести задачи на поиск медианы в потоке однообразных данных. В таких алгоритмах нет необходимости сортировать весь список значений и использование кучи будет эффективно.

В-третьих, куча встречается в графовых алгоритмах. Классический пример — алгоритм Дейкстры, где на каждом шаге нужно быстро получить вершину с минимальной текущей оценкой расстояния.
### Заключение
Зафиксируем определение:
> Куча — это структура данных, которая поддерживает частичный порядок и позволяет эффективно получать минимальный или максимальный элемент, поэтому она подходит для задач с приоритетом.

## Задачи: сравнение подходов
### Задача для стека
Посмотрим на задачу, где необходимо проверить корректности последовательности скобок. Стек здесь является эффективным решением:
- последовательно обрабатывая полученные значения мы:
- открывающую скобку кладём в стек;
- закрывающую сверяем с вершиной стека;
- если всё сошлось и стек в конце пуст — строка корректна.
### Задача для кучи
Вторая задача — найти `k` минимальных элементов. Можно было бы полностью отсортировать массив но сортировка переставляет вообще все элементы, хотя нам нужны только несколько лучших.

Куча позволяет решить задачу более эффективно:
- либо построить min-heap из всех элементов и извлечь первые `k`,
- либо поддерживать кучу фиксированного размера.

## Заключение урока

Давайте подведём итог. У нас было две структуры данных:
- **стек** — для работы по принципу LIFO, вложенности, отката и обратного порядка обработки,
- **кучу** — для задач, где важен приоритет и быстрый доступ к минимальному или максимальному элементу.
Обе структуры часто реализуются поверх массива, но реализуют разную логику доступа к данным.

Стек отвечает на вопрос: **какой элемент был добавлен последним?**

Куча отвечает на вопрос: **какой элемент сейчас лучший по приоритету?**
